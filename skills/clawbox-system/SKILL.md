# clawbox-system — 产品体系总纲

一句话:**我们在做"陪伴孩子学习成长的个人 AI"**;当前主战场 **Tapio(K12 英语私教)单点破局**,ClawBox 平台与 PawClass 大脑为其供弹药。

## 四层结构(要什么读哪)
| 层 | 文件 | 装什么 | 变化频率 |
|---|---|---|---|
| 决策 | `strategy.md` | 使命→发展路线→方向判断(带日期的决策记录) | 月级 |
| 作战 | `roadmap.md` | Demo→产品阶段计划 + 北极星 + 差距清单 | 周级 |
| 地图 | `portfolio.md` | 产品矩阵 + 每条线一页纸(状态/仓库/深度认知入口) | 周级 |
| 操作 | `runbook.md` | 每条线的部署 SOP + 查问题入口 | 随架构变 |
| 迭代 | `progress.md` | 进行中的事 + 待验证 + 已排除(git 表达不了的) | 每次会话 |

## 体系铁律
1. **嘴脑分离**:交互端(嘴)只管体验;学情事实只住 PawClass(脑),任何端不得私建学情存储。
2. **孩子的体验 > 开发效率 > 技术优雅;安全 > 功能;陪伴感 > 响应速度**(全线适用)。
3. **一份事实一个家**:深度认知住各项目的认知 skill/memory,本 skill 只做体系层与索引,永不复制细节。
4. 工程细则(≤500行/无死代码/验证要证据)见本仓 `dev-standards` 与各项目 CLAUDE.md。

## 迭代仪式(让体系"系统地"往前滚)
- **开工**:读 `progress.md`(30 秒恢复状态)+ **值守**:curl https://tapio.teachclaw.app/healthz(非 200 即最高优先级)→ 若做方向决策,先读 `strategy.md` 的"方向判断"。
- **干活**:动哪条线,按 `portfolio.md` 该线的"深度认知入口"进入(clawbox skill / Tapio memory / 各 CLAUDE.md)。
- **收工**:把状态变化写回 `progress.md`;方向被推翻/确认 → 更新 `strategy.md` 并标日期;新线/砍线 → `portfolio.md`。
- **纪律**:每篇 ≤100 行;超了说明该下沉到项目层。

## 维护
方向决策必须带日期与一句话依据;三个月未被引用的段落删掉或下沉。
