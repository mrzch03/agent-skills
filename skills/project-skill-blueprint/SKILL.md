# project-skill-blueprint — 项目认知 skill 的骨架

一个"项目认知 skill"= 让任何 agent / 新人**接手即懂**这个项目的认知移植包。它高内聚(只装这个项目的事)、用完即丢(项目结束整包归档)、能排障、能交接。本蓝图给结构与约定;照着建一份即可。

## 三层架构(依赖单向,引用不复制)

```
元事实(knowledge/lessons/troubleshooting)   ← 无依赖、被依赖的客观事实;表格化、按领域分文件
   ↑ 引用
工作单元(工作区根 issues/,skill 外)          ← bug/需求,一事一文件;与 forge issue、评审文档同步
   ↑ 指路
交接(handoff.md)                            ← 主线指针 + 在飞清单;一事一两行,完结即删行
```

- **元事实**:客观、慢变、带日期/依据。**优先表格行**(事实 | 依据/日期 | 备注),可枚举的必须表格,叙事性的(链路串讲)留短散文。永不引用工作单元。
- **工作单元**:bug=现象/排查/定位/修复/证据;需求=方案/上线记录。行文一针见血。本地文件=在飞期间的工作真相;开工建 forge issue(GitHub/GitLab),关键节点同步评论,**关闭后 forge 是归档**——可泛化认知先分流进元事实,本地文件留一句话摘要或删。技术方案本地 md 为源,评审平台文档(飞书/Notion)同步,token/链接记文件头。
- **交接**:接手先读。只指路不装货;历史=git log(commit 引 issue 号)+ 已关 forge issue,**永不留第二份**。
- ⚠️ **反模式:追加型进度日志**(progress.md 记流水)——只写不读的仪式,单条越写越长、有用信息靠 grep 碰运气,且截胡了本该进元事实各页的知识。已废弃,别再建。

## 目录骨架(按需取舍,不必全有)
```
SKILL.md           入口:一句话定位 + 铁律 + 层级 + 目录(要干啥→读哪)+ 维护规则
knowledge/         元事实(慢变,表格化分领域)
  concepts.md        术语表 + 核心不变量
  architecture.md    链路 + 职责切分 + 代码锚点
  contracts.md       对外契约(协议/接口/错误码,带对齐日期)
  environment.md     拓扑 / 权限矩阵 / 固定测试资产 / 环境怪癖
lessons/           经验(增长型,也是元事实)
  heuristics.md      决策法则(动手前过一遍,每条带案例锚点)
  case-studies.md    大案复盘(现象→路径→根因→教训,≤10 行/案)
troubleshooting/   排障(按症状索引)
  symptoms-*.md      症状 → 先查 → 历史根因(grep 报错就跳到根因)
  commands.md        现成命令(查日志/查库/常用诊断)
handoff.md         交接:主线指针 + 在飞清单(完结即删行)
tools/             诊断 CLI(把"每次都重写的查询"固化成脚本)
../issues/         工作单元在工作区根(跨仓共享),skill 里只放指针
```

## SKILL.md 五段约定
1. **一句话定位**:这个项目是什么、现在主战场在哪。
2. **铁律(违反即事故)**:跨组件 × 静默变错的硬约束,编号列出。改它前先肃然起敬。
3. **层级**:上面的三层声明 + 本工作区 issues/ 路径。
4. **目录(你要干什么 → 读哪)**:"任务→文件"导航,别让人翻。
5. **维护(谁改谁更新,人和 AI 同规)**:什么触发更新哪个文件,**同一工作单元内改完**——这是 skill 不腐烂的关键。

## 五条维护纪律(决定它会不会变成没人敢动的大文件)
- **一个事实一个家,永不复制**;通用规范引 `dev-standards`,不在本 skill 重抄。
- **每篇 ≤ 100 行**;超了按主题(领域)拆,用时才加载(progressive disclosure)。
- **改代码的同一工作单元更新对应 skill**(skill 即真相,不留滞后)。
- **慢变进元事实,进展/证据进 issues/ 对应文件,handoff 只动指针行**;别混。
- **完结即清**:issue 关了删清单行、归档 forge;历史永远只有一份。

## 怎么起步
1. 复制 `references/skeleton/` 到你的 skill 目录,改 frontmatter 的 name/description。
2. 先写 SKILL.md 的"一句话定位 + 铁律 + 层级";其余文件**有内容再建**,别先摆空壳。
3. 工作区根建 `issues/`(README 收录模板 + 同步契约),第一个在飞工作单元就位后 handoff.md 指过去。
4. 排障时:症状沉淀进 `troubleshooting/`,可泛化的法则进 `lessons/heuristics.md`,跑过的诊断固化进 `tools/`。

> 参考实例:秒播 `clone-anchor-unify`(元事实五篇+handoff+根 issues/)是本蓝图的第一个完整实例。
